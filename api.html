class wormtable.Database(homedir, db_name)¶

The superclass of database objects. Databases are located in a home directory and are backed by a two files: a database file and an xml metadata file.

close()¶

Closes this database object, freeing underlying resources.

delete()¶

Deletes the DB and metadata files for this database.

exists()¶

Returns True if this Database exists.

finalise_build()¶

Move the build file to its final location and write the metadata file.

get_cache_size()¶

Returns the cache size for this database in bytes.

get_db_build_path()¶

Returns the path of the file used to build the database.

get_db_file_size()¶

Returns the size of the database file in bytes.

get_db_name()¶

Returns the db name of this database object.

get_db_path()¶

Returns the path of the permanent file used to store the database.

get_homedir()¶

Returns the home directory for this database object.

get_ll_object()¶

Returns the low-level object that this Database is a facade for.

get_metadata_path()¶

Returns the path of the file used to store metadata for the database.

get_open_mode()¶

Returns the mode that this database is opened in, WT_READ or WT_WRITE. If the database is not open, return None.

is_open()¶

Returns True if this database is open for reading or writing.

open(mode)¶

Opens this database in the specified mode. Mode must be one of ‘r’ or ‘w’.

read_metadata(filename)¶

set_cache_size(cache_size)¶

Sets the cache size for this database object to the specified value. If cache_size is a string, it can be suffixed with K, M or G to specify units of Kibibytes, Mibibytes or Gibibytes.

bin_widths()¶

Returns the list of bin widths in this index.

Builds this index. If progress_callback is not None, invoke this calback after every callback_rows have been processed.

class wormtable.IndexCounter(index)¶

A counter for Indexes, based on the collections.Counter class. This class is a dictionary-like object that represents a mapping of the distinct keys in the index to the number of times those keys appear.

class wormtable.Table(homedir)¶

The main storage table class.

add_char_column(name, description='', num_elements=0)¶

Creates a new character column with the specified name, description and number of elements. If num_elements=0 then the column can hold variable length strings; otherwise, it can contain strings of a fixed length only.

add_column(name, description, element_type, size, num_elements)¶

Creates a new column with the specified name, description, element type, element size and number of elements.

add_float_column(name, description='', size=4, num_elements=1)¶

Creates a new float column with the specified name, element size (in bytes) and number of elements. If num_elements=0 then the column can hold a variable number of elements. Only 4 and 8 byte floats are supported by wormtable; these correspond to the usual float and double types.

add_id_column(size=4)¶

Adds the ID column with the specified size in bytes.

add_int_column(name, description='', size=2, num_elements=1)¶

Creates a new integer column with the specified name, element size (in bytes) and number of elements. If num_elements=0 then the column can hold a variable number of elements.

add_uint_column(name, description='', size=2, num_elements=1)¶

Creates a new unsigned integer column with the specified name, element size (in bytes) and number of elements. If num_elements=0 then the column can hold a variable number of elements.

append(row)¶

Appends the specified row to this table.

append_encoded(row)¶

Appends the specified row to this table.

close()¶

Closes this Table.

columns()¶

Returns the list of columns.

cursor(columns, index=None)¶

Returns a cursor over the rows in this database, retreiving the specified columns. If index is provided, the cursor will iterate over the rows in the order defined by the index.

Columns must be a list of column identifiers, or Column instances.

get_column(col_id)¶

Returns a column corresponding to the specified id. If this is an integer, we return the column at this position; if it is a string we return the column with the specified name.

get_fixed_region_size()¶

Returns the size of the fixed region in rows. This is the minimum size that a row can be; if there are no variable sized columns in the table, then this is the exact size of each row.

get_metadata()¶

Returns an ElementTree instance describing the metadata for this Table.

indexes()¶

Returns an open index on the this table. Supports the contextmanager protocol.

set_metadata(tree)¶

Sets up this Table to reflect the metadata in the specified xml ElementTree.
